# Migration from Existing Systems

# Possible Cases

# Case 1: Startup/Starting from Scratch

You can install and start using the system.

# Case 2: Startup - It's ok to request users to set new password

  1. Install OmniLogin
  2. Migrate your users details from old database talbe to omnilogin.users table
  3. Inform your users to reset their password.
  4. This project currently uses node-php-password (opens new window) for password encryption

Note: You can fork repo and add your custom password encryption & verification logic.

# Case 3: Enterprise Company - Overnight Migration

  1. Install OmniLogin
  2. Migrate your users details from old database talbe to omnilogin.users table

# Case 4: Enterprise Company - Migration - Recommended

  1. Install OmniLogin
  2. Start using for internal employees
  3. Wait for developer to understand all basic features available in omnilogin
  4. Integrate with existing CRM and check integration happening smoothly
  5. Test all the features
  6. Switch OmniLogin as primary Signup/Login Source

# Migration FAQ

# Can I use my existing database

Ans: Yes, you can use existing database, OmniLogin will create few more tables

# Existing users table can be used. so no issues of synchronization at the time of migration

Ans: Yes, you can connect existing users table. so customer login from your existing login system or OmniLogin, both uses same central datasource
